Magic Box

Summary

Exploring the possibilities

We drew inspiration from objects in nature and in pop culture/media that stood out to us as 'Magical' and then narrowed down the list to a number of elementary concepts that we might be able to embody using mechatronics elements. (Listed above)

 

We dropped the idea of the monster book because emulating that interaction satisfactorily would have required too many elements and we didn't have the time

Translating elements to electronics

Once we had decided what elements we wanted in the box, we translated them into interactions or mechanisms that could be achieved using components that were readily available and within our budget.

 

Notice there's two Arduino MCUs in the box, this is because the PWM library in the Arduino IDE overrides any other timing based function and makes it incredibly difficult to work with any other kind of Library

The Box

The result was a Light gray box, that let just the right amount of light through so as to not give away the discrete nature of the 'moving' LEDs. When the box is picked up, the lights go from a lazy crawl to a quick, excited motion. There is also a switch on the bottom of the box that stays closed while the box is down. When the box is lifted, it opens and a signal generator creates a gentle humming/vibration in the box by using sine waves running through surface transducers

Opening/locking mechanism

The box also had no obvious way to open it. The only way it would open is if someone passed a small magnet (Easily hidden between the fingers) across the middle of the box.

 

This was achieved using a hall sensor that controlled a servo driven Lock-Prop Mechanism right under the lid

When at the 0 degree position, the mechanism hooks into a latch on the lid, and when it turns to 180degrees, the hook releases the latch and a cam props up the lid
